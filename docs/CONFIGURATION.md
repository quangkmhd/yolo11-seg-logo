# Configuration Guide

The project uses a hierarchical configuration system to manage settings for training, inference, and data processing.

## 1. Environment Variables (`.env`)

The recommended way to configure the project is by creating a `.env` file in the root directory. Copy the template from `.env.example`.

| Variable | Description | Default |
| :--- | :--- | :--- |
| `SOURCE_DATA_DIR` | Path to raw images and Labelme JSONs | `.../data_logo` |
| `OUTPUT_DATA_DIR` | Path where the YOLO dataset will be built | `.../yolo_dataset` |
| `MODEL_CHECKPOINT` | Default model file for inference / Gradio | `best.pt` |
| `EPOCHS` | Number of training iterations | `100` |
| `BATCH` | Batch size for training | `16` |
| `DEVICE` | Computation device (`0`, `1`, or `cpu`) | `0` |

---

## 2. Config Class (`src/logo_seg/config.py`)

If environment variables are not provided, the system falls back to the defaults defined in the `Config` class. 

### Why use `config.py`?
- **Immutability**: Ensures paths are consistent across different modules (Data, Model, App).
- **Auto-Discovery**: Automatically resolves the project root directory relative to the file location.
- **Directory Management**: Contains helper methods to ensure required directories exist before execution.

---

## 3. Dataset Specification (`data.yaml`)

This file is automatically generated by the `main.py data convert` command. It tells the YOLO core engine where to find processed images and label names.

```yaml
path: /absolute/path/to/dataset
train: images/train
val: images/val
nc: 1
names: ['logo']
```

---

## 4. Hardware Optimization

For production workloads, you can adjust the `DEVICE` setting:
- **Single GPU**: `DEVICE=0`
- **Multi-GPU**: `DEVICE=0,1,2`
- **CPU Only**: `DEVICE=cpu` (Not recommended for training)
